01

Modelo Entidad-Relación

GA6-220501096-AA1-EV02

CONCEPTO

¿Qué es un modelo E-R?

Es un dibujo que representa cómo se organizan los datos antes de programarlos.

Sirve para ponerse de acuerdo entre el cliente, el equipo y el desarrollador sobre qué información guarda el sistema y cómo se conecta.

  • Entidad: una "cosa" del mundo real (Cliente, Carro, Factura).
  • Atributo: una característica de la entidad (placa, modelo, marca).
  • Relación: cómo se conectan las entidades (un Cliente tiene Carros).

CARDINALIDAD

¿Cuántos con cuántos?

La cardinalidad le pone números a la relación. Es obligatoria en la evidencia.

1 : 1

Uno con uno.

Persona ↔ Cédula

1 : N

Uno con muchos.

Cliente → muchos Carros

N : M

Muchos con muchos.

Estudiantes ↔ Cursos

💡 Tip: en tu diagrama deben aparecer estos números (1, N, M) en cada línea que une dos entidades.

02

Sentencias NoSQL

GA6-220501096-AA1-EV03

CAMBIO DE MUNDO

SQL vs NoSQL en 30 segundos

SQL

Tablas · Filas · Columnas

  • Estructura fija
  • Esquema definido
  • MySQL, Postgres, Oracle
VS

NoSQL

Colecciones · Documentos · JSON

  • Estructura flexible
  • Cada documento es distinto
  • MongoDB, Redis, Cassandra

En esta evidencia trabajamos con MongoDB: cada "registro" es un objeto JSON.

PASO 1 · MODELAR

El objeto carro en JSON

El enunciado pide modelar un carro con 6 campos. Así se ve: 


{
  "placa":         "ABC123",
  "numero_serie":  "SN-0001-2024",
  "modelo":        2022,
  "marca":         "Toyota",
  "kilometraje":   45000,
  "tipo":          "Sedán"
}

💡 Tip: los textos van entre comillas, los números no. tipo puede ser "Sedán", "SUV", "Pickup", "Hatchback"…

PASO 2 · CREAR E INSERTAR

Colección + 5 documentos


// Crear la colección (en EV04 se llamará "parque")
db.createCollection("carros")

// Insertar 5 carros de un solo golpe
db.carros.insertMany([
  { placa:"ABC123", numero_serie:"SN-0001", modelo:2022,
    marca:"Toyota",  kilometraje:45000, tipo:"Sedán" },
  { placa:"DEF456", numero_serie:"SN-0002", modelo:2020,
    marca:"Chevrolet", kilometraje:78000, tipo:"SUV" },
  { placa:"GHI789", numero_serie:"SN-0003", modelo:2023,
    marca:"Mazda",   kilometraje:12000, tipo:"Hatchback" },
  { placa:"JKL012", numero_serie:"SN-0004", modelo:2019,
    marca:"Renault", kilometraje:95000, tipo:"Pickup" },
  { placa:"MNO345", numero_serie:"SN-0005", modelo:2024,
    marca:"Kia",     kilometraje:5000,  tipo:"Sedán" }
])
PASO 3 · CONSULTAR Y ACTUALIZAR

Ver y modificar los datos


// Consultar todos los documentos
db.carros.find()

// Consultarlos formateados (más legible)
db.carros.find().pretty()

// Buscar un carro específico
db.carros.findOne({ placa: "ABC123" })

// Actualizar el kilometraje de un carro
db.carros.updateOne(
  { placa: "ABC123" },
  { $set: { kilometraje: 47500 } }
)

// Actualizar varios a la vez (todos los Toyota)
db.carros.updateMany(
  { marca: "Toyota" },
  { $set: { revisado: true } }
)

COMANDOS · PARTE 1 (PASOS 1–4)

Setup, inserción y actualización


// 1) Crear y usar la base de datos
use parqueDB

// 2) Crear la colección "parque"
db.createCollection("parque")

// 3) Insertar 5 documentos
db.parque.insertMany([
  { placa:"ABC123", numero_serie:"SN-0001", modelo:2022, marca:"Toyota",    kilometraje:45000, tipo:"Sedán"     },
  { placa:"DEF456", numero_serie:"SN-0002", modelo:2020, marca:"Chevrolet", kilometraje:78000, tipo:"SUV"       },
  { placa:"GHI789", numero_serie:"SN-0003", modelo:2023, marca:"Mazda",     kilometraje:12000, tipo:"Hatchback" },
  { placa:"JKL012", numero_serie:"SN-0004", modelo:2019, marca:"Renault",   kilometraje:95000, tipo:"Pickup"    },
  { placa:"MNO345", numero_serie:"SN-0005", modelo:2024, marca:"Kia",       kilometraje:5000,  tipo:"Sedán"     }
])

// 4) Actualizar primer y último registro
db.parque.updateOne({ placa:"ABC123" }, { $set: { kilometraje: 50000 } })  // primero
db.parque.updateOne({ placa:"MNO345" }, { $set: { kilometraje: 8000  } })  // último
COMANDOS · PARTE 2 (PASOS 5–8)

Listar, borrar y buscar


// 5) Listar la colección completa
db.parque.find().pretty()

// 6) Borrar el TERCER documento (placa GHI789)
db.parque.deleteOne({ placa: "GHI789" })

// 7) Listar otra vez para mostrar que ya son 4
db.parque.find().pretty()

// 8) Obtener un documento según número de placa
db.parque.findOne({ placa: "DEF456" })
