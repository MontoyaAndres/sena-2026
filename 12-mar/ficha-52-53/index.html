4 campos requeridos

Nombres · Apellidos · Cédula · Fecha de nacimiento. Cada campo con su etiqueta visible arriba.

Tipo de dato

Cédula: solo números. Fecha: selector de calendario o formato DD/MM/AAAA para evitar errores.

Los 4 campos deben tener el mismo ancho, fuente y altura para que la pantalla se vea ordenada.

Color de error

El rojo se usa SOLO para errores. No lo uses en ningún otro lugar de la aplicación.

Redacción del mensaje

Di QUÉ pasó y QUÉ debe hacer el usuario. Evita mensajes técnicos como "Error 401".

Paleta de colores · Máximo 3 colores

Los 3 colores deben verse coherentes en todas las pantallas

#2563EB
Color principal
Botones, títulos, barras
#F3F4F6
Color neutro
Fondos, campos, tarjetas
#DC2626
Color de alerta
Solo para errores
💡 Regla 60 / 30 / 10: El color neutro ocupa el 60% (fondos), el color principal el 30% (botones, encabezados) y el color de alerta solo el 10%, únicamente cuando hay un error.

Los 4 conceptos que debes investigar

Usa al menos 3 fuentes: libros, artículos o sitios académicos

# Concepto ¿Qué buscar?
1 Definición de calidad de software ¿Qué significa que un software sea de calidad? Norma ISO 25010
