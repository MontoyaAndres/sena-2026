Evidencia 01
Foro temático

Licenciamiento de software

Pregunta 1 · ¿Qué es software?

Base conceptual antes de hablar de licencias

El software es el conjunto de programas, instrucciones y datos que permiten a un computador realizar tareas específicas. A diferencia del hardware, el software no se puede tocar; es intangible.

⚙️

Software de sistema

Controla el hardware. Ej: Windows, Linux, macOS

📱

Software de aplicación

Resuelve tareas del usuario. Ej: Word, Chrome, WhatsApp

🛠️

Software de desarrollo

Sirve para crear otros programas. Ej: VS Code, Eclipse

Pregunta 2 · ¿Qué es una licencia de software?

Es un contrato legal entre el creador y el usuario

Una licencia de software es un contrato que define los derechos y restricciones de uso de un programa. Establece qué puede hacer el usuario: instalarlo, copiarlo, modificarlo o redistribuirlo.

✅

¿Qué permite?

Usar el software en ciertos dispositivos · Hacer copias de seguridad · Acceder a actualizaciones · Usar en un plazo determinado

🚫

¿Qué prohíbe?

Distribuir sin autorización · Modificar el código fuente · Usar en más equipos de los permitidos · Revender la licencia

Pregunta 3 · Tipos de licencia de software

Existen varios tipos según el nivel de libertad que otorgan

Propietaria / Comercial

El código fuente es privado. El usuario paga por usarlo pero no puede modificarlo. Ej: Windows, Microsoft Office.

Restrictiva

Software Libre (GPL)

El código es abierto. Se puede usar, copiar, modificar y redistribuir libremente. Ej: Linux, LibreOffice.

Libre

Freeware

Gratuito para usar, pero el código fuente sigue siendo privado. No se puede modificar. Ej: Adobe Reader, Zoom.

Gratis · Sin código

Shareware

Se puede probar por un tiempo limitado o con funciones reducidas. Luego requiere pago. Ej: WinRAR.

Prueba / Pago

Más tipos de licencia

Otras categorías importantes que debes conocer

Tipo ¿Qué permite? Ejemplo
Open Source (MIT / Apache) Usar y modificar libremente, incluso en proyectos comerciales React, Angular
Creative Commons Para contenidos creativos: se define si se puede modificar o usar comercialmente Wikipedia
Dominio Público Sin restricciones. Cualquiera puede usarlo, modificarlo y venderlo SQLite
Freemium / SaaS Versión básica gratis, funciones avanzadas de pago, como servicio en la nube Spotify, Notion

Pregunta 4 · ¿Cuáles son las más adecuadas?

Depende del contexto: desarrollo, empresa o educación

🎓

Para aprender / estudiar

Software libre y Open Source. Permiten ver el código, modificarlo y practicar sin restricciones ni costos.

🏢

Para proyectos empresariales

MIT / Apache. Permiten usar código abierto en productos comerciales con muy pocas restricciones.

💻

Para uso personal

Freeware o Freemium. Son gratuitas, fáciles de instalar y cubren las necesidades cotidianas.

Ficha técnica

Recolección de información · NTC ISO 9000

¿Qué es una ficha técnica?

Documento estandarizado para registrar información de un producto de software

Una ficha técnica es un documento estructurado que reúne las características principales de un producto o fuente de información. Permite identificar, registrar y consultar datos de forma organizada y coherente.

🔍

Identifica

Permite reconocer claramente de dónde vino la información y quién la produjo.

📊

Organiza

Estructura la información en campos definidos para que sea fácil de leer y consultar.

✅

Garantiza calidad

Siguiendo la norma NTC ISO 9000, asegura que los datos sean confiables y trazables.

Referencia · Norma NTC ISO 9000

Sistema de Gestión de Calidad — base para diseñar tu ficha

La NTC ISO 9000 establece los fundamentos y vocabulario de los Sistemas de Gestión de Calidad. Busca garantizar que los procesos sean consistentes, documentados y mejorables.

📌

¿Por qué usarla en la ficha?

Porque la norma exige que toda información recolectada tenga: nombre, origen, fecha, responsable y criterios de calidad verificables.

🔗

Principios clave que aplican

Trazabilidad de la información · Documentación clara · Identificación de fuentes · Control de registros.

Ejemplo · Estructura de la ficha técnica

Tu ficha debe tener al mínimo estos campos

📋 FICHA TÉCNICA — Recolección de Información
Nombre del software
Ej: Mozilla Firefox
Versión
Ej: 120.0
Tipo de licencia
Ej: Mozilla Public License (Open Source)
Fabricante / Autor
Ej: Mozilla Foundation
Fecha de consulta
Ej: 12/03/2026
Fuente de información
Ej: https://www.mozilla.org
Descripción general
Ej: Navegador web de código abierto multiplataforma…
Criterios de calidad
Ej: Fuente oficial, información verificable, vigente

Pasos para elaborar la ficha técnica

Sigue esta secuencia para construir tu documento

1

Estudia la norma NTC ISO 9000

Revisa los conceptos de trazabilidad, registro y control de documentos que aplican al diseño de fichas.

2

Define los campos de la ficha

Identifica qué información necesitas recolectar: nombre, versión, licencia, autor, fuente, descripción, criterios de calidad.

3

Diseña la ficha

Puedes hacerla en Excel (tabla) o en Word/PDF. Debe verse ordenada, con etiquetas claras y espacio suficiente para cada campo.

4

Llena la ficha con una fuente real

Selecciona un software concreto (por ejemplo, uno con licencia libre) y completa todos los campos de tu ficha.
