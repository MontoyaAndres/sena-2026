Licenciamiento de Software

Evidencia GA2-220501094-AA1-EV01

Foro temático

💡 ¿Qué es software?

El software es el conjunto de programas, instrucciones y reglas informáticas que permiten a un dispositivo (computador, celular, servidor) ejecutar tareas específicas. Es la parte lógica e intangible de un sistema informático, a diferencia del hardware que es la parte física.
⚙️

Software de sistema

Sistemas operativos, controladores, firmware (Windows, Linux, Android).

📱

Software de aplicación

Programas para el usuario final (Word, Excel, navegadores, apps).

🛠️

Software de desarrollo

Herramientas para crear software (IDE, compiladores, frameworks).

📜 ¿Qué es una licencia de software?

Una licencia de software es un contrato legal entre el autor o propietario (licenciante) y el usuario (licenciatario), que establece los derechos, obligaciones y condiciones bajo los cuales se puede usar, copiar, modificar o distribuir un programa.

¿Qué define una licencia?

✅ Qué SÍ se puede hacer

Uso personal, comercial, número de instalaciones, copias de respaldo, modificaciones.

❌ Qué NO se puede hacer

Redistribución no autorizada, ingeniería inversa, revender, compartir credenciales.

⏰ Duración

Temporal, permanente, por suscripción mensual o anual.

💰 Condiciones económicas

Precio, forma de pago, renovación, actualizaciones incluidas o no.

⚖️ ¿Por qué son importantes las licencias?

🛡️ Protección legal

Protegen los derechos de autor y la propiedad intelectual del creador del software.

💼 Cumplimiento normativo

Evitan sanciones legales, multas y demandas por uso indebido de software.

🔐 Seguridad

Garantizan que el software sea auténtico, con soporte y actualizaciones.

💡 Claridad

Definen claramente los derechos y deberes de ambas partes.

📌 En Colombia: el uso de software sin licencia es un delito según la Ley 603 de 2000 y la Ley 1915 de 2018 (derechos de autor). Las empresas deben reportar el cumplimiento en sus informes de gestión.

🔖 Tipos de licencias de software

Existen varios tipos de licencia, clasificados principalmente por el nivel de libertad que otorgan al usuario y por su modelo económico.

Veremos las 8 más importantes

1️⃣ Licencia propietaria o comercial

El código fuente es privado. El usuario paga para usar el software bajo condiciones estrictas.

✓ Ventajas

  • Soporte técnico oficial
  • Actualizaciones garantizadas
  • Calidad y estabilidad comprobada

✗ Desventajas

  • Costo elevado
  • Dependencia del fabricante
  • No se puede modificar el código

Ejemplos: Microsoft Windows Adobe Photoshop AutoCAD SAP

2️⃣ Licencia de código abierto (Open Source)

Permite ver, modificar y distribuir el código fuente libremente, bajo ciertas condiciones.

✓ Ventajas

  • Generalmente gratuito
  • Comunidad activa de desarrollo
  • Transparencia y seguridad
  • Personalización total

✗ Desventajas

  • Soporte no siempre oficial
  • Curva de aprendizaje
  • Compatibilidad variable

Sublicencias populares: GPL MIT Apache BSD

Ejemplos: Linux, LibreOffice, GIMP, MySQL, Apache, VLC.

3️⃣ Software Libre (Free Software)

Definido por la Free Software Foundation, garantiza las 4 libertades esenciales al usuario.

0 Usar

Libertad de ejecutar el programa para cualquier propósito.

1 Estudiar

Libertad de estudiar cómo funciona y adaptarlo (acceso al código).

2 Distribuir

Libertad de redistribuir copias para ayudar a otros.

3 Mejorar

Libertad de mejorar el programa y publicar las mejoras.

Nota: Software libre ≠ software gratis. "Free" se refiere a libertad, no al precio.

4️⃣ Freeware

Software gratuito para usar, pero el código fuente es cerrado. No se puede modificar ni redistribuir comercialmente.

📌 Características

  • Uso sin costo
  • Sin acceso al código
  • Puede tener restricciones de uso comercial

💡 Ejemplos

Skype Adobe Reader WhatsApp Zoom (versión básica)

5️⃣ Shareware

Software de evaluación gratuita por un período limitado o con funciones reducidas. Después se debe pagar para seguir usándolo.

⏱ Trialware

Uso completo por tiempo limitado (ej: 30 días).

🧩 Demo

Funciones limitadas de forma permanente.

💸 Adware

Gratis pero con publicidad incluida.

Ejemplos: WinRAR, algunas versiones de antivirus, juegos demo.

6️⃣ Licencia OEM (Original Equipment Manufacturer)

Software que viene preinstalado en un equipo nuevo y está vinculado al hardware. No se puede transferir a otro equipo.

✓ Ventajas

  • Más económica
  • Ya instalada y configurada

✗ Desventajas

  • Atada al equipo original
  • Soporte limitado del fabricante del PC

Ejemplo: Windows que viene con un computador Lenovo o HP.

7️⃣ Licencia por volumen (Volume License)

Permite instalar el mismo software en múltiples equipos de una organización con una sola clave o contrato.

🏢 Ideal para

  • Empresas medianas y grandes
  • Instituciones educativas
  • Entidades del gobierno

💰 Beneficios

  • Descuentos por cantidad
  • Administración centralizada
  • Soporte empresarial

Ejemplos: Microsoft Volume Licensing, Adobe Creative Cloud for Teams.

8️⃣ Licencia por suscripción (SaaS)

Software as a Service: se paga de forma periódica (mensual o anual) por el uso del software, generalmente en la nube.

✓ Ventajas

  • Sin instalación local
  • Actualizaciones automáticas
  • Acceso desde cualquier lugar
  • Menor inversión inicial

✗ Desventajas

  • Requiere conexión a internet
  • Costo acumulado a largo plazo
  • Dependencia del proveedor

Ejemplos: Microsoft 365, Google Workspace, Netflix, Spotify, Salesforce, Canva Pro.

🎯 ¿Cuáles son las más adecuadas?

Depende del contexto, el presupuesto y las necesidades:

🏠 Uso personal

Freeware u Open Source: económicos, suficientes para tareas cotidianas.

🎓 Educación

Software Libre y licencias académicas: promueven aprendizaje y son de bajo costo.

🏢 PYMES

SaaS o Open Source: baja inversión inicial, flexibilidad, escalabilidad.

🏛️ Grandes empresas

Volumen o Propietaria: soporte robusto, control centralizado, cumplimiento legal.

🚀 Realidad actual del licenciamiento

Hoy en día predomina el modelo SaaS (suscripción) por encima de las licencias perpetuas. Las empresas migran hacia la nube para reducir costos iniciales y obtener flexibilidad.

📈 Crecimiento del SaaS

Microsoft 365, Google Workspace, Adobe CC, Salesforce dominan el mercado empresarial.

🌍 Auge del Open Source

Linux domina servidores; Android, WordPress, Kubernetes impulsan la industria.

🤖 IA y nuevas licencias

Surgen licencias específicas para modelos de IA (OpenRAIL, Apache 2.0 modificadas).

⚖️ Cumplimiento estricto

Auditorías de software más frecuentes, mayor conciencia sobre piratería.

Fichas Técnicas

Evidencia GA2-220501094-AA1-EV02

Diseño para la recolección de información

📋 ¿Qué es una ficha técnica?

Una ficha técnica es un documento estructurado que reúne las características, especificaciones y atributos de un producto de software. Permite el registro, identificación y consulta organizada de las fuentes de información.

Funciones principales:

📑 Documentar

Reúne de forma clara las especificaciones del software desarrollado.

🔍 Identificar

Permite reconocer rápidamente el producto y sus fuentes.

📊 Comparar

Facilita la comparación entre diferentes soluciones o versiones.

✅ Validar

Sirve como evidencia de cumplimiento de requisitos y normatividad.

💡 ¿Por qué son importantes?

  • Estandarización: unifican la forma en que se recolecta y presenta la información.
  • Trazabilidad: permiten seguir el historial y cambios del producto.
  • Comunicación: facilitan el diálogo entre equipos técnicos, comerciales y clientes.
  • Toma de decisiones: brindan datos concretos para decidir sobre el producto.
  • Cumplimiento normativo: evidencian el apego a estándares como ISO 9000.
📌 En propuestas técnicas de TI: la ficha técnica es un anexo fundamental que respalda la seriedad y formalidad de la propuesta.

📏 Norma NTC ISO 9000

La NTC ISO 9000 es la norma técnica colombiana que establece los fundamentos y el vocabulario de los Sistemas de Gestión de Calidad (SGC). Forma parte de la familia ISO 9000, que es reconocida internacionalmente.

🎯 Objetivo

Establecer el marco conceptual y terminología común para los SGC en cualquier organización.

🌐 Alcance

Aplica a productos y servicios, independientemente del sector (incluye software).

🎯 7 Principios de gestión de calidad (ISO 9000)

1 Enfoque al cliente

Cumplir y superar expectativas.

2 Liderazgo

Dirección unificada y comprometida.

3 Compromiso de las personas

Personal competente y motivado.

4 Enfoque a procesos

Gestionar actividades interrelacionadas.

5 Mejora

Enfoque permanente en mejorar.

6 Toma de decisiones basada en evidencia

Datos y análisis como base.

7 Gestión de relaciones

Relaciones con proveedores y partes interesadas.

🧩 Elementos clave de una ficha técnica

📌 Información general

  • Nombre del producto/software
  • Versión
  • Código o identificador
  • Fecha de elaboración
  • Responsable / autor
  • Empresa desarrolladora

⚙️ Especificaciones técnicas

  • Lenguaje de programación
  • Plataforma / sistema operativo
  • Requisitos de hardware
  • Dependencias y librerías
  • Base de datos

📜 Información legal

  • Tipo de licencia
  • Condiciones de uso
  • Vigencia
  • Derechos de autor

💰 Información comercial

  • Precio / modelo de pago
  • Garantía y soporte
  • Canales de distribución

✅ Calidad (ISO 9000)

  • Indicadores de calidad
  • Criterios de aceptación
  • Proceso de validación

🔎 Características de las fuentes de información

Al diseñar una ficha técnica, se debe considerar el tipo de fuente a consultar:

📚 Fuentes primarias

Información original y directa: entrevistas, encuestas, observación, documentación propia del software.

📖 Fuentes secundarias

Análisis o interpretación de fuentes primarias: libros, artículos, reportes técnicos, manuales.

🌐 Fuentes terciarias

Compilaciones de fuentes secundarias: bases de datos, directorios, enciclopedias.

💾 Fuentes digitales

Repositorios, páginas web oficiales, documentación técnica en línea, foros especializados.

✔ Criterio clave: toda fuente debe ser confiable, actual y verificable.

✨ Buenas prácticas al elaborar la ficha

🎯 Claridad

Usar lenguaje técnico preciso pero comprensible.

📐 Estructura

Organizar por secciones claras y lógicas.

📊 Completitud

Incluir todos los campos necesarios, sin excesos.

🔄 Actualización

Mantener versiones y fechas visibles.

🎨 Presentación

Formato profesional, logos, colores institucionales.

📎 Referencias

Citar fuentes consultadas correctamente.
