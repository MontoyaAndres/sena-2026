¿Qué es front-end?

Es la parte visible del software

Incluye pantallas, menús, botones, formularios, colores, textos, imágenes y comportamientos que usa el usuario.

Un buen front-end no solo “se ve bonito”: también debe ser claro, fácil de usar y coherente con el proyecto.

<html> estructura
.css presentación
JS() interacción

HTML, CSS y JavaScript en una sola idea

HTML

Define la estructura.

Títulos, párrafos, listas, enlaces, imágenes, formularios y secciones.

CSS

Define la apariencia.

Colores, tamaños, márgenes, tipografía, diseño responsive y distribución.

JavaScript

Define la interacción.

Validaciones, botones, menús, cambios dinámicos y eventos del usuario.

Usabilidad: que el usuario entienda sin sufrir

Buenas prácticas

  • Textos claros y botones visibles.
  • Menú fácil de encontrar.
  • Colores coherentes y buen contraste.
  • Diseño adaptable a celular y computador.
  • Mensajes de error comprensibles.

Evitar

  • Páginas desordenadas.
  • Textos pequeños o difíciles de leer.
  • Botones sin función.
  • Imágenes pesadas o sin relación.
  • Archivos sueltos sin carpeta organizada.

Checklist antes de entregar

Revisión técnica

  • El ZIP abre correctamente.
  • El index.html funciona.
  • Las imágenes cargan.
  • El CSS está enlazado.
  • El JS no genera errores visibles.

Revisión formativa

  • La evidencia corresponde al código solicitado.
  • Se entiende el propósito del diseño.
  • Hay relación con el proyecto.
  • Se aplican conceptos de usabilidad.
  • La presentación es ordenada.

Actividad rápida en clase

Trabajo por equipos

En 20 minutos, cada equipo define una pantalla principal del proyecto y responde:

1. ¿Qué verá el usuario?

Secciones, títulos, imágenes y formularios.

2. ¿Qué componentes necesita?

Menú, tarjetas, botones, tablas, login, etc.

3. ¿Qué interacción tendrá?

Validaciones, alertas, cambios de estado o navegación.
