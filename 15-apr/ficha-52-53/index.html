Maquetación de la interfaz gráfica en HTML
EV04

¿Qué es maquetar?

Maquetar es organizar y combinar texto, imágenes, colores, tamaños y animaciones en una página web usando HTML, CSS y JavaScript, para que luzca bien tanto en computadores como en celulares y tablets.
🖼️

Composición visual

Defines dónde va cada elemento: títulos, imágenes, botones, videos.

🎨

Colores y tipografías

Eliges la paleta de colores y los estilos de letra que representan la marca.

📐

Responsive design

La página se adapta a cualquier pantalla: PC, tablet o móvil.

EV04

¿Cómo lo hago? — Paso a paso

  • 1 Estudia los requisitos del cliente — ¿Qué necesita mostrar la web? ¿Quiénes son los usuarios?
  • 2 Boceta el diseño — Dibuja en papel o en Figma cómo lucirá cada página antes de escribir código.
  • 3 Escribe el HTML — Estructura el contenido con etiquetas semánticas (<header>, <main>, <footer>).
  • 4 Aplica CSS / Framework — Usa Bootstrap, Tailwind o Foundation para estilos rápidos y responsivos.
  • 5 Agrega interactividad — Con JavaScript añade animaciones, validaciones o efectos visuales.
EV04

Herramientas recomendadas

⚡

Bootstrap

El framework más popular. Grid de 12 columnas, componentes listos y docs increíbles.

🌬️

Tailwind CSS

Clases utilitarias en el HTML. Total control del diseño sin salir del archivo.

🏗️

Foundation

Alternativa robusta orientada a proyectos profesionales y accesibilidad.

Mapa de Navegación
EV05

¿Qué es un mapa de navegación?

Es un árbol jerárquico que muestra todas las páginas del sitio web y cómo están conectadas entre sí. Es como el plano de un edificio: antes de construir, sabes cuántos pisos hay y cómo se llega a cada cuarto.
📋

Lineal

Páginas una tras otra en secuencia. Ideal para tutoriales o formularios paso a paso.

🌳

Jerárquico

Estructura de árbol: inicio → secciones → subsecciones. El más común en sitios web.

🕸️

Web / Compuesto

Cualquier página puede enlazar a cualquier otra. Usado en wikis o portales grandes.

EV05

¿Cómo lo construyo?

  • 1 Lista todo el contenido — ¿Qué información debe mostrar el sitio? Escríbela toda.
  • 2 Agrupa por categorías — Junta contenido relacionado: Inicio, Sobre nosotros, Servicios, Contacto…
  • 3 Define la jerarquía — ¿Qué va primero? ¿Qué es submenú de qué?
  • 4 Dibuja el árbol — Usa papel, Figma, Miro o cualquier herramienta de diagramas.
  • 5 Valida con otros — Muéstraselo a alguien que no sepa del proyecto. Si lo entiende, ¡vas bien!

Usabilidad y Accesibilidad en Apps Móviles
EV06

Usabilidad vs. Accesibilidad

Concepto ¿Qué significa? Ejemplo práctico
🎯 Usabilidad Que la app sea fácil e intuitiva de usar para cualquier persona. Botón grande de "Comprar" que se encuentra de inmediato.
♿ Accesibilidad Que la app pueda usarse por personas con discapacidades visuales, motrices, etc. Textos alternativos en imágenes para lectores de pantalla.
💡 Una app puede ser usable pero NO accesible. Lo ideal es que sea ambas cosas a la vez.
EV06

Tipos de aplicaciones móviles

📱

Nativa

Construida específicamente para un SO. Swift/SwiftUI para iOS, Kotlin para Android. Máximo rendimiento.

🔀

Híbrida

Un solo código para varios SO. Usa tecnologías web (HTML/JS) envueltas en un contenedor nativo. Ej: Ionic, React Native.

💻

Independiente (Desktop)

Corre en el sistema operativo del PC sin necesidad de navegador ni conexión. Ej: apps de escritorio con Electron.

10 reglas de oro de usabilidad

2. Correspondencia con el mundo real
3. Control y libertad del usuario
4. Consistencia y estándares
5. Prevención de errores
6. Reconocimiento antes de recordar
7. Flexibilidad y eficiencia
8. Estética y diseño minimalista
9. Ayuda a reconocer y recuperarse de errores
10. Ayuda y documentación
