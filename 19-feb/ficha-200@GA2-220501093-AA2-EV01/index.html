Ficha 200 — Evidencia EV01

Modelo de Dominio (UML)

Vamos a hacer 2 dibujos (diagramas) y un documento en PDF.

Diagrama de clases Diagrama de paquetes Casos de uso

¿Qué tengo que entregar?

  • Un PDF con:
  • ✅ Diagrama de clases (dominio)
  • ✅ Diagrama de paquetes
  • ✅ Casos de uso (plantilla)

La extensión es libre (puede ser corto si está bien hecho).

¿Qué es “modelo de dominio”?

Es una lista de “cosas importantes” del sistema y cómo se conectan.

  • Cosas = clases (Usuario, Pedido, Producto)
  • Conexiones = relaciones
  • Cuántas conexiones = cardinalidad

Paso 1: lee requisitos

  • ¿Qué hace el sistema?
  • ¿Quién lo usa? (actores)
  • Reglas importantes: “debe”, “no puede”, “mínimo”, “máximo”

Sin requisitos, no sabemos qué dibujar.

Paso 2: saca las clases (las “cosas”)

Busca sustantivos

  • Usuario
  • Curso
  • Inscripción
  • Pago

No uses

  • Botón
  • Pantalla
  • Formulario

Paso 3: une clases con relaciones

Preguntas rápidas:

  • ¿Usuario tiene pedidos?
  • ¿Pedido tiene productos?
  • ¿Curso tiene estudiantes?

Dibuja líneas entre clases cuando “tengan que ver” una con otra.

Paso 4: pon cardinalidad (cuántos)

  • 1..1 = exactamente uno
  • 0..1 = opcional (puede no tener)
  • 0..* = cero o muchos
  • 1..* = uno o muchos

Ejemplo: Usuario 1 — Pedido 0..*

Paso 5: herencia (solo si aplica)

Herencia = una clase “es un tipo de” otra.

Ejemplo: Persona → Estudiante / Profesor

Si no lo necesitas, no la uses.

Paso 6: diagrama de paquetes

Paquetes = carpetas para ordenar el sistema.

  • Usuarios
  • Productos
  • Pedidos
  • Pagos

Paso 7: casos de uso (plantilla)

  • Nombre del caso de uso
  • Actor
  • Objetivo
  • Pasos (flujo principal)
  • Errores / alternativas

Haz 3 a 6 casos de uso principales.

Herramientas para dibujar

  • diagrams.net (draw.io)
  • StarUML
  • Visual Paradigm
  • Lucidchart

Luego exportas y lo pones en el PDF.

Checklist final

  • ✅ Diagrama de clases con cardinalidades
  • ✅ Diagrama ordenado y legible
  • ✅ Diagrama de paquetes
  • ✅ Casos de uso con plantilla
  • ✅ Todo en PDF (con portada)
