Ficha 52 y 53 — Evidencia EV03

Diagrama de Despliegue (UML)

Es un dibujo para mostrar dónde corre el sistema y cómo se conecta.

Nodos Software Conexiones

¿Qué tengo que entregar?

  • Un documento (normalmente PDF) con:
  • ✅ Portada
  • ✅ Introducción
  • ✅ Diagrama de despliegue
  • ✅ Conclusión

Debe servir para el caso de estudio y el proyecto.

¿Qué es un diagrama de despliegue?

Es como un mapa del sistema:

  • Máquinas / lugares donde corre (nodos)
  • Programas que se instalan ahí (artefactos)
  • Cables / internet que los conectan (enlaces)

3 palabras clave

Nodo

Un “lugar”

  • PC
  • Servidor
  • Celular
  • Nube

Artefacto

Un “programa”

  • App web
  • API
  • Base de datos

Y los enlaces son las conexiones (Internet / red).

Paso 1: define tu arquitectura (simple)

  • ¿Tienes web, backend y base de datos?
  • ¿Se usa en celular o en navegador?
  • ¿Está en nube o en un PC/servidor?

Arquitectura = las partes principales del sistema.

Paso 2: dibuja los nodos

Ejemplo típico:

  • Nodo 1: Cliente (PC o Celular)
  • Nodo 2: Servidor (API / app)
  • Nodo 3: Base de datos

Primero dibuja “dónde corre”, luego qué corre ahí.

Paso 3: pon los programas dentro (artefactos)

  • Dentro del nodo Cliente: Navegador o App móvil
  • Dentro del Servidor: Backend / API
  • Dentro de BD: PostgreSQL / MySQL (la base de datos)

Artefacto = lo que se instala / ejecuta en el nodo.

Paso 4: conecta con enlaces

  • Cliente ↔ Servidor: HTTPS (internet)
  • Servidor ↔ Base de datos: conexión interna (red)

Las flechas/líneas muestran quién habla con quién.

Requisitos no funcionales (en fácil)

  • Seguridad: usar HTTPS, no exponer la base de datos
  • Velocidad: cache o buen servidor (si aplica)
  • Disponibilidad: copias/backup (si aplica)

Tu diagrama debe “tener sentido” con estos puntos.

Herramientas para hacer el diagrama

  • diagrams.net (draw.io)
  • StarUML
  • Visual Paradigm
  • Lucidchart

Exporta y lo metes en el documento.

Cómo debe verse el documento

  • Portada: nombre, evidencia, ficha, fecha
  • Introducción: qué es el diagrama y para qué sirve
  • Diagrama: nodos + artefactos + enlaces
  • Conclusión: qué aprendiste y cómo cumple requisitos

Checklist final

  • ✅ Nodos claros (cliente, servidor, BD…)
  • ✅ Programas dentro de cada nodo
  • ✅ Líneas de conexión correctas
  • ✅ Seguridad básica (HTTPS, BD no pública)
  • ✅ Documento con portada + intro + conclusión
