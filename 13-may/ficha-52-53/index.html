Antes de empezar

¿Qué es SQL?

SQL (Structured Query Language) es el idioma con el que le hablamos a una base de datos.

Con SQL podemos:

  • Crear tablas y definir su forma
  • Guardar información (insertar)
  • Buscar y filtrar datos
  • Modificar o eliminar registros

Piensa en SQL como…

Una conversación con una bibliotecaria muy organizada: tú le pides "muéstrame todos los libros de terror publicados después de 2010" y ella te trae la lista exacta.

La diferencia: en vez de hablar, escribimos comandos.

Dos familias de comandos

DDL vs DML

Familia ¿Para qué sirve? Comandos típicos
DDL
Data Definition Language 		Define la estructura: crea y modifica tablas, columnas y restricciones. CREATE   ALTER   DROP   TRUNCATE
DML
Data Manipulation Language 		Manipula los datos que están dentro de las tablas. INSERT   SELECT   UPDATE   DELETE
Regla mental: DDL cambia el edificio; DML cambia los muebles dentro del edificio.

Destrezas con DDL y DML: el taller "Libreta"

Vamos a construir, llenar y consultar una pequeña agenda de contactos. Ocho pasos, ocho comandos clave.

1

Crear la tabla libreta

Tres columnas: nombre, domicilio y teléfono. 


CREATE TABLE libreta (
  nombre    VARCHAR(20),
  domicilio VARCHAR(30),
  telefono  VARCHAR(11)
);
VARCHAR(n) significa "texto de hasta n caracteres". Se usa para nombres, direcciones, etc. El teléfono va como texto porque no vamos a hacer matemáticas con él.
DDLCREATE TABLE
2

Ver las tablas existentes

Confirmamos que libreta se creó bien. 


SHOW TABLES;

Resultado esperado: aparece libreta en el listado de tablas de la base de datos.

MySQLVerificación
3

Describir la estructura

¿Qué columnas tiene? ¿De qué tipo? ¿Aceptan nulos? 


DESCRIBE libreta;
DESCRIBE nos muestra una "radiografía" de la tabla: cada columna, su tipo, si admite NULL, si tiene llave primaria, valor por defecto, etc.
DESCRIBEMetadatos
4

Insertar los primeros registros


INSERT INTO libreta (nombre, domicilio, telefono)
VALUES ('Alberto Mores', 'Colon 123',      '4234567');

INSERT INTO libreta (nombre, domicilio, telefono)
VALUES ('Juan Torres',   'Avellaneda 135', '4458787');
Cada INSERT guarda una fila. Fíjate en el orden: los valores deben corresponder a las columnas que listamos arriba.
DMLINSERT
5

Mostrar todos los registros


SELECT * FROM libreta;

El asterisco * significa "todas las columnas". Es la consulta más usada del mundo.

Tip de profesional: en producción evitamos SELECT * y pedimos solo las columnas necesarias. Es más rápido y más claro.
DMLSELECT
6

Actualizar los datos insertados


UPDATE libreta
SET    domicilio = 'Colon 456'
WHERE  nombre = 'Alberto Mores';

UPDATE libreta
SET    telefono = '4400000'
WHERE  nombre = 'Juan Torres';
¡Atención! Un UPDATE sin WHERE modifica todas las filas de la tabla. Siempre, siempre, siempre: filtra con WHERE.
DMLUPDATE
7

Insertar 5 registros más


INSERT INTO libreta (nombre, domicilio, telefono) VALUES
  ('María López',    'Bolívar 78',     '4112233'),
  ('Carlos Ruiz',    'San Martín 12',  '4778899'),
  ('Lucía Gómez',    'Sarmiento 540',  '4334455'),
  ('Diego Pérez',    'Mitre 901',      '4556677'),
  ('Sofía Castro',   'Belgrano 230',   '4998877');
Tip: con INSERT INTO ... VALUES (...), (...), (...) podemos insertar varias filas en un solo comando. Más rápido que cinco INSERT separados.
DMLINSERT múltiple
8

Contar cuántos registros hay


SELECT COUNT(*) AS total_contactos
FROM   libreta;

Si hicimos todo bien, el resultado debería ser 7: los 2 iniciales + los 5 nuevos.

COUNT(*) es una función de agregación: en vez de devolver filas, devuelve un número. Veremos más (SUM, AVG, MAX, MIN) en próximas sesiones.
DMLAgregación

Diseñar la estructura de la BD del proyecto

Ahora pensamos en tu proyecto formativo: ¿qué tablas necesita?, ¿cómo se relacionan?, ¿qué reglas debe respetar?

Bloque 1 · Tipos de datos

Elige el tipo correcto para cada columna

Tipo Para qué sirve Ejemplo
INT Números enteros (edades, cantidades, IDs) edad INT
DECIMAL(p,e) Números con decimales exactos (dinero, precios) precio DECIMAL(10,2)
VARCHAR(n) Texto de longitud variable (nombres, correos) nombre VARCHAR(50)
TEXT Textos largos (descripciones, comentarios) descripcion TEXT
DATE Fechas (año-mes-día) fecha_nac DATE
DATETIME Fechas con hora creado_en DATETIME
BOOLEAN Verdadero/Falso (activo, eliminado, etc.) activo BOOLEAN
Bloque 2 · Restricciones

Llave Primaria (PRIMARY KEY)

Es el identificador único de cada fila. Como tu número de cédula:

  • No se puede repetir
  • No puede ser NULL
  • Solo hay una por tabla

CREATE TABLE estudiante (
  id_estudiante INT PRIMARY KEY AUTO_INCREMENT,
  nombre        VARCHAR(60) NOT NULL,
  correo        VARCHAR(80) UNIQUE
);
AUTO_INCREMENT hace que la base de datos genere el ID automáticamente: 1, 2, 3, 4… sin que tú te preocupes.
Bloque 2 · Restricciones

Llave Foránea (FOREIGN KEY)

Conecta dos tablas. Es como decir: "esta matrícula pertenece a ese estudiante". 


CREATE TABLE matricula (
  id_matricula  INT PRIMARY KEY AUTO_INCREMENT,
  id_estudiante INT NOT NULL,
  curso         VARCHAR(80),
  fecha         DATE,

  CONSTRAINT fk_matricula_estudiante
    FOREIGN KEY (id_estudiante)
    REFERENCES estudiante(id_estudiante)
    ON DELETE RESTRICT
    ON UPDATE CASCADE
);
La FK garantiza la integridad referencial: no puedes matricular a un estudiante que no existe.
Bloque 2 · Restricciones

Otras restricciones útiles

NOT NULL

La columna no puede quedar vacía. Útil para datos obligatorios.

UNIQUE

El valor no se puede repetir, pero sí puede ser NULL. Ideal para correos.

DEFAULT

Si no envías valor, usa uno por defecto. Ej: activo BOOLEAN DEFAULT TRUE.

CHECK

Valida una condición. Ej: CHECK (edad >= 0).

AUTO_INCREMENT

Genera IDs automáticos. Casi siempre va junto a la llave primaria.

INDEX

Acelera las búsquedas en columnas que se consultan con frecuencia.

Flujo recomendado

Mini ejemplo completo

Una BD pequeña con sus restricciones


CREATE TABLE categoria (
  id_categoria INT PRIMARY KEY AUTO_INCREMENT,
  nombre       VARCHAR(50) NOT NULL UNIQUE
);

CREATE TABLE producto (
  id_producto  INT PRIMARY KEY AUTO_INCREMENT,
  nombre       VARCHAR(80) NOT NULL,
  precio       DECIMAL(10,2) NOT NULL CHECK (precio >= 0),
  stock        INT DEFAULT 0,
  id_categoria INT NOT NULL,
  creado_en    DATETIME DEFAULT CURRENT_TIMESTAMP,

  CONSTRAINT fk_producto_categoria
    FOREIGN KEY (id_categoria) REFERENCES categoria(id_categoria)
);

Aquí tienes: PK, FK, NOT NULL, UNIQUE, DEFAULT y CHECK — todo en menos de 15 líneas.

Generar el script final de tu base de datos

Convertimos el diseño visual en código SQL ejecutable: el "modelo físico" listo para correr en un servidor.

Concepto clave

Modelo conceptual → lógico → físico

1

Conceptual

¿Qué entidades hay? ¿Cómo se relacionan? Aún sin pensar en SQL.

2

Lógico

Tablas, atributos, llaves… pero sin un motor específico todavía.

3

Físico

Código SQL real, listo para MySQL: tipos de datos exactos, motor InnoDB, charset, etc.

La EV03 te pide exactamente el modelo físico — un archivo .sql que cualquiera pueda ejecutar para reconstruir tu base de datos desde cero.
Paso a paso en Workbench

Cómo generar y exportar el script

  • Abre tu diagrama E-R en MySQL Workbench
  • Menú Database → Forward Engineer…
  • Selecciona las opciones: DROP objects before CREATE, generar inserts si los tienes
  • Revisa el script generado y copia o guarda como archivo .sql
  • Comprime ese .sql en un ZIP y entrégalo
Tip: abre el .sql en un editor de texto antes de entregarlo y verifica que tenga CREATE DATABASE, todos los CREATE TABLE y las FOREIGN KEY.
Así se ve un script exportado

Fragmento de ejemplo


-- -----------------------------------------------------
-- Schema proyecto_formativo
-- -----------------------------------------------------
CREATE SCHEMA IF NOT EXISTS `proyecto_formativo`
  DEFAULT CHARACTER SET utf8mb4;
USE `proyecto_formativo`;

-- -----------------------------------------------------
-- Table `categoria`
-- -----------------------------------------------------
CREATE TABLE IF NOT EXISTS `categoria` (
  `id_categoria` INT NOT NULL AUTO_INCREMENT,
  `nombre`       VARCHAR(50) NOT NULL,
  PRIMARY KEY (`id_categoria`),
  UNIQUE INDEX `uq_categoria_nombre` (`nombre` ASC)
) ENGINE = InnoDB;

Workbench te genera todos los comentarios, los IF NOT EXISTS y el motor — tú solo lo entregas.
