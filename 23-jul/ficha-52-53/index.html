03 Concepto

¿Qué es una API?

Es la puerta por la que dos programas se hablan. Uno envía una petición y recibe una respuesta.

GET
/api/productos/15
Send
Petición método + endpoint + datos
Respuesta código de estado + JSON
04 Herramienta

¿Qué es Postman?

Un cliente que envía peticiones HTTP a tu API y te muestra la respuesta, sin escribir código de frontend. Ideal para probar cada endpoint por separado.

  • Elegir el método (GET, POST…)
  • Escribir la URL del endpoint
  • Enviar datos en el Body
  • Ver la respuesta y el código de estado
05 Motivación

¿Por qué hacer testing?

Funciona

Confirmar que cada endpoint responde y devuelve los datos esperados.

Responde bien

Verificar los códigos de estado correctos en cada caso.

Maneja errores

Comprobar qué pasa con datos inválidos o recursos que no existen.

07 Cómo leer la respuesta

Códigos de estado

2xx

200 OK · 201 Creado. Todo salió bien.

4xx

400 Petición mala · 404 No existe. Error del cliente.

5xx

500 Error interno. Falló el servidor.

Cada prueba debe mostrar el código que devuelve el endpoint.
