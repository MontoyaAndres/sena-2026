Sección 02

Referentes Técnicos

Las condiciones técnicas definen el entorno necesario para que la solución de software funcione correctamente

¿Qué son los Referentes Técnicos?

Son el conjunto de especificaciones y condiciones tecnológicas que debe cumplir tanto el entorno de desarrollo como el entorno de producción para que la solución de software opere de manera óptima y cumpla con los requisitos del cliente.

🖥️

Hardware

CPU, RAM, almacenamiento

💿

Software

OS, BD, frameworks

🌐

Red

Ancho de banda, protocolos

🔐

Seguridad

Cifrado, backups, roles

Categorías de Especificación Técnica

🖥️

Infraestructura Hardware

Servidores, estaciones de trabajo, dispositivos de red (routers, switches), equipos de respaldo y UPS.

💿

Stack de Software

Lenguaje de programación, framework de desarrollo, servidor web (Apache/Nginx), motor de base de datos y herramientas de CI/CD.

🌐

Infraestructura de Red

Topología de red, protocolos usados (HTTP/S, TCP/IP), VPN si se requiere, firewall y monitoreo de red.

🔐

Seguridad e Integridad

Política de contraseñas, cifrado en tránsito y en reposo, plan de respaldo (backup), control de acceso basado en roles (RBAC).

Sección 03

Condiciones Económicas

La estimación económica proyecta el costo total del desarrollo y permite evaluar la viabilidad del proyecto

¿Por qué Estimar los Costos?

📊

Tomar decisiones

Saber si el proyecto es viable antes de invertir tiempo y recursos

📋

Planificar recursos

Definir cuántos desarrolladores, en cuánto tiempo y con qué herramientas

🤝

Negociar con el cliente

Sustentar el presupuesto presentado al cliente con datos reales

💡 Una estimación incorrecta puede hacer que un proyecto fracase por falta de presupuesto o que se pierda el contrato por cotizar demasiado caro.

Métodos de Estimación de Software

Puntos de Función

Function Points (FP)

Mide la funcionalidad del software desde la perspectiva del usuario. Cuenta entradas, salidas, consultas, archivos e interfaces externas.

COCOMO II

Modelo Constructivo de Costos

Estima esfuerzo y tiempo según el tamaño del software (líneas de código) y factores de costo del equipo y plataforma.

Juicio de Expertos

Analogía y Delphi

Se basa en experiencia previa con proyectos similares. Varios expertos estiman de forma independiente y se consolida un consenso.

Planning Poker

Story Points (Ágil)

En metodologías ágiles, el equipo asigna puntos de historia a las tareas del backlog para estimar el esfuerzo relativo.

Estructura de Costos del Proyecto

Sección 04

Cronograma del Proyecto

Las fases del proyecto estructuran el trabajo en el tiempo y permiten controlar el avance

Fases del Desarrollo de Software

2 semanas

Fase 1 · Levantamiento de Requisitos

Entrevistas con el cliente, análisis de necesidades, definición del alcance y elaboración del documento de especificación.

3 semanas

Fase 2 · Diseño del Sistema

Arquitectura de software, diseño de base de datos, prototipos de interfaz (wireframes) y definición de APIs.

8 semanas

Fase 3 · Desarrollo e Implementación

Codificación del backend y frontend, integración de módulos, revisiones de código y pruebas unitarias.

3 semanas

Fase 4 · Pruebas y Control de Calidad

Pruebas funcionales, de rendimiento, de seguridad y de aceptación con el usuario final.

2 semanas

Fase 5 · Despliegue y Capacitación

Puesta en producción, migración de datos, manual de usuario y entrenamiento al personal.

💡 Consejos para tu Informe

🎯 Sé específico con el contexto: describe la empresa del caso de estudio (sector, tamaño, necesidad) antes de proponer las especificaciones técnicas.
📐 Justifica cada especificación: no basta con decir «necesita 16 GB de RAM»; explica por qué esa cantidad es necesaria para ese proyecto.
📊 Usa tablas y gráficos: un presupuesto en tabla es más claro que párrafos con números. Los lectores agradecen la información organizada visualmente.
🔢 Detalla el costo por ítem: cantidad × valor unitario = total. Así el cliente entiende exactamente en qué se invierte cada peso.
📎 Cita tus fuentes: precios de servidores, salarios de referencia y licencias deben venir de fuentes verificables (sitios oficiales, portales de empleo, etc.).
✅ Revisa el número de páginas: mínimo 7, máximo 10 hojas de contenido. ¡No cuentes la portada ni los anexos!
