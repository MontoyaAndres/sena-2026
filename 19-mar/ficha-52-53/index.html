Sección 01

Conceptos Clave

¿De qué hablamos cuando decimos «usabilidad» y «accesibilidad»?

¿Qué es la Usabilidad?

La usabilidad es la medida en que un sistema puede ser usado por usuarios específicos para alcanzar metas específicas con efectividad, eficiencia y satisfacción en un contexto de uso determinado. (ISO 9241-11)

✅ Efectividad

El usuario logra su objetivo de forma completa y correcta

⚡ Eficiencia

Se usa el menor tiempo y esfuerzo posible

😊 Satisfacción

El usuario se siente cómodo y a gusto con la app

¿Qué es la Accesibilidad?

La accesibilidad garantiza que todas las personas, independientemente de sus capacidades físicas, sensoriales o cognitivas, puedan percibir, entender, navegar e interactuar con una aplicación sin barreras.

👁️

Discapacidad visual

Lectores de pantalla, alto contraste, texto alternativo en imágenes

👂

Discapacidad auditiva

Subtítulos, transcripciones, alertas visuales

🖐️

Discapacidad motriz

Navegación por teclado, botones grandes, comandos de voz

🧠

Discapacidad cognitiva

Lenguaje simple, íconos claros, flujos predecibles

Sección 02

Reglas de Usabilidad

Las 10 Heurísticas de Jakob Nielsen (1994)

Heurísticas 1 – 5

01 Visibilidad del estado del sistemaEl sistema informa siempre al usuario sobre lo que está ocurriendo (indicadores de carga, confirmaciones).
02 Correspondencia con el mundo realUsa lenguaje familiar al usuario; evita términos técnicos internos del sistema.
03 Control y libertad del usuarioOfrece «deshacer» y «rehacer»; el usuario puede salir de estados no deseados fácilmente.
04 Consistencia y estándaresLas convenciones de plataforma se respetan; palabras iguales significan cosas iguales.
05 Prevención de erroresDiseña para evitar que ocurran problemas antes que simplemente reportarlos.

Heurísticas 6 – 10

06 Reconocer antes que recordarObjetos, acciones y opciones visibles; el usuario no debe memorizar información entre pantallas.
07 Flexibilidad y eficienciaLos usuarios expertos pueden usar atajos; la interfaz se adapta a novatos y avanzados.
08 Diseño estético y minimalistaSolo muestra información relevante; cada elemento extra compite con los importantes.
09 Reconocer y recuperarse de erroresLos mensajes de error indican claramente qué salió mal y cómo resolverlo.
10 Ayuda y documentaciónAunque la app sea intuitiva, el usuario debe poder acceder a ayuda cuando la necesite.
Sección 03

Reglas de Accesibilidad

WCAG 2.1 y el modelo POUR

WCAG — Pautas de Accesibilidad

Web Content Accessibility Guidelines (WCAG) son las normas internacionales del W3C que definen cómo hacer contenido digital accesible. Se organizan en 3 niveles:

Nivel A · Mínimo Nivel AA · Recomendado Nivel AAA · Óptimo
P
Perceptible

La info debe poder percibirse

O
Operable

Los componentes son operables

U
Comprensible

Información y UI comprensibles

R
Robusto

Compatible con tecnologías asistivas

Reglas Prácticas de Accesibilidad

A Texto alternativoToda imagen debe tener un atributo alt descriptivo para lectores de pantalla.
B Contraste de colorRelación mínima de 4.5:1 entre texto y fondo (nivel AA).
C Navegación por tecladoToda funcionalidad debe ser accesible sin mouse; uso de Tab y Enter.
D Tamaño de fuente ajustableEl texto debe poder ampliarse hasta 200% sin perder contenido ni funcionalidad.
E Etiquetas en formulariosCada campo debe tener su etiqueta label asociada correctamente.
F Mensajes de error clarosIndicar qué campo tiene error y cómo corregirlo, no solo con color.
Sección 04

Tipos de Aplicaciones

Independientes vs Web — ¿cuáles son las diferencias?

Aplicación Independiente vs Web

💻 Aplicación Independiente (Desktop)
Se instala directamente en el sistema operativo
Funciona sin conexión a Internet
Mayor acceso a recursos del hardware (GPU, archivos)
Actualizaciones manuales o automáticas por instalador
Ejemplos: Microsoft Word, Photoshop, VLC
Tecnologías: Java, .NET, C++, Python (Tkinter/PyQt)
🌐 Aplicación Web
Se ejecuta en un navegador; no requiere instalación
Requiere conexión a Internet (normalmente)
Acceso limitado al hardware por seguridad del navegador
Actualizaciones instantáneas en el servidor
Ejemplos: Gmail, Google Docs, Trello, Figma
Tecnologías: HTML, CSS, JavaScript, React, Angular

Características de una App Independiente

📦

Instalación local

El ejecutable (.exe, .dmg, .deb) se descarga e instala en el dispositivo del usuario

🔌

Modo offline

Funciona sin red; los datos se almacenan localmente en el disco

⚙️

Alto rendimiento

Aprovecha al máximo la CPU, memoria y GPU del equipo local

🔐

Seguridad local

Los datos sensibles pueden no salir del dispositivo; menor exposición en red

🖥️

Integración con OS

Accede a impresoras, cámara, micrófono y sistema de archivos directamente

🔄

Ciclo de actualización

El usuario debe descargar e instalar nuevas versiones manualmente o con instalador

Características de una App Web

🌍

Acceso universal

Disponible desde cualquier dispositivo con navegador y conexión a Internet

🚀

Despliegue inmediato

Una actualización en el servidor llega a todos los usuarios al instante

📱

Multiplataforma

Funciona en Windows, macOS, Linux, Android, iOS sin cambios de código

🤝

Colaboración en tiempo real

Múltiples usuarios pueden trabajar simultáneamente sobre los mismos datos

💰

Menor coste de distribución

No hay distribución de instaladores; el usuario accede con una URL

🛡️

Limitaciones de seguridad

El sandbox del navegador restringe el acceso al hardware por protección

💡 Buenas Prácticas para Ambos Tipos

🎨 Diseño consistente: Usa paletas de colores y tipografías coherentes en toda la aplicación para no confundir al usuario.
📐 Jerarquía visual: Lo más importante debe verse primero. Tamaño, color y posición guían la atención.
⚡ Retroalimentación inmediata: Siempre confirma las acciones del usuario: loaders, mensajes de éxito/error, animaciones.
♿ Pruebas de accesibilidad: Usa herramientas como WAVE, Lighthouse o axe para validar el cumplimiento WCAG.
👥 Pruebas con usuarios reales: El mayor test de usabilidad es que personas reales usen tu app sin ayuda.
