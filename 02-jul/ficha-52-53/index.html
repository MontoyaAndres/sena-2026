Idea base

¿Qué es React?

Una biblioteca de JavaScript para construir interfaces de usuario dividiéndolas en componentes: piezas pequeñas, independientes y reutilizables.

Piensa en bloques de LEGO: un botón, una tarjeta, un menú. Los armas una vez y los reutilizas en toda la app. 
// Un componente = una pieza de la interfaz
function Tarjeta() {
  return (
    <div className="tarjeta">
      <h2>Hola SENA</h2>
    </div>
  );
}

EV01 · Pregunta 1

React vs JSX

React

Es la biblioteca: la lógica que crea componentes, maneja el estado y actualiza la pantalla.

JSX

Es la sintaxis: una forma de escribir la interfaz que se parece a HTML, pero vive dentro de JavaScript. 

// JSX: se ve como HTML…
const saludo = <h1>Hola</h1>;

// …pero por dentro es React puro:
const saludo = React.createElement(
  'h1', null, 'Hola'
);
En una frase: React es el motor; JSX es el idioma cómodo con el que le hablas.

EV01 · Pregunta 2

¿Qué son las clases en React?

Una forma de crear componentes usando clases de JavaScript (extends React.Component) con un método render() que devuelve la interfaz.

Componente de clase

class Boton extends React.Component {
  render() {
    return <button>
      {this.props.texto}
    </button>;
  }
}

Estilo moderno (funciones + Hooks) 

function Boton({ texto }) {
  return <button>{texto}</button>;
}
Tip: hoy se prefieren los componentes de función con Hooks, pero conocer las clases te ayuda a leer código existente.

Principales eventos de React

Los eventos permiten reaccionar a lo que hace el usuario. Se escriben en camelCase y reciben una función.

onClick — al hacer clic onChange — al escribir onSubmit — al enviar un formulario onMouseOver — al pasar el mouse onKeyDown — al presionar una tecla 
function Contador() {
  const manejarClic = () => alert('¡Hiciste clic!');
  return <button onClick={manejarClic}>Presióname</button>;
}

EV01 · Pregunta 4

Mapa conceptual de React

React
Biblioteca de JavaScript para construir interfaces con componentes
Componentes
Piezas reutilizables de la interfaz (funciones o clases)
JSX
Sintaxis parecida a HTML para describir la vista
Props
Datos que un componente recibe desde afuera
Estado (state)
Datos internos que cambian y redibujan la vista
Eventos
Reaccionan a las acciones del usuario
Virtual DOM
Actualiza la pantalla de forma rápida y eficiente

Elige y justifica tus componentes

Según las características de tu software, describe qué componentes de React usarás. Cada elección necesita su porqué.

Ejemplos de componentes

  • Formulario de registro / login
  • Tabla o lista de datos
  • Barra de navegación (menú)
  • Tarjetas de producto o perfil

La justificación responde…

  • ¿Qué función cumple en el proyecto?
  • ¿Por qué es reutilizable?
  • ¿Qué datos maneja (props / estado)?
