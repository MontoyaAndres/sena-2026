¿de qué hablamos?

Las piezas de la web con Java

Servlet

Clase Java que vive en el servidor y responde a las peticiones del navegador. Es la "lógica" detrás de cada pantalla.

GET vs POST

GET pide datos y los manda visibles en la URL. POST envía datos en el cuerpo: ideal para formularios y contraseñas.

JSP — JavaServer Pages

Páginas que mezclan HTML con código Java. Sirven para mostrar al usuario la respuesta que prepara el servlet.

Formulario HTML

La puerta de entrada de los datos. Su action apunta al servlet y su method define GET o POST.

cómo se ve en código

Del formulario al servlet

registro.html 
<form action="registro" method="post">
  <input type="text" name="nombre">
  <input type="email" name="correo">
  <button>Registrar</button>
</form>
RegistroServlet.java 
@WebServlet("/registro")
public class RegistroServlet extends HttpServlet {
  // recibe el envío POST del formulario
  protected void doPost(HttpServletRequest req,
                         HttpServletResponse resp) {
    String nombre = req.getParameter("nombre");
    req.setAttribute("nombre", nombre);
    req.getRequestDispatcher("ok.jsp")
       .forward(req, resp);
  }
}

El proyecto debe crearse con versionamiento y combinar formularios HTML, servlets (GET y POST) y elementos JSP.
